1. OOP 단계별 접근
	1) step1: 기본 변수, 개별 선언
	2) step2: 배열 활용, 데이터 그룹화
	3) step3: 클래스 도입, 객체 생성
	4) step4: 객체 배열

2. 클래스가 필요한 이유
	1) 세상은 클래스와 객체로 이루어져 있음
		- 클래스와 객체라는 개념이 중요함
		
	2) 문제: 쇼핑몰 물건 구매하는 프로그램 만들기
		- 요구사항
			- 첫 번째 상품의 이름은 "상품 1 - 상품명: 아이폰17 | 가격: 1400000 | 카테고리: "전자제품" | 재고: 50개
			- 첫 번째 상품의 이름은 "상품 2 - 상품명: 나이키 신발 | 가격: 150000 | 카테고리: "신발" | 재고: 30개
			
		- 요구사항 추가됨
			- 상품이 늘어남
				-> 배열을 사용함
			-배열 사용의 한계
				- 한 상품의 데이터가 4개의 배열에 나누어서 할당
				- 데이터를 변경할 때 매우 조심히 작업해야 함
				- 특정 상품이 데이터를 변경할 때 실수할 가능성이 맹 높음
				- 사람이 관리하기에는 좋은 코드가 아님
	3) 클래스 도입
		- 클래스를 사용하여 상품이라는 개념을 만들고,
		  상품별로 이름, 가격, 재고, 카테고리를 관리함
		- 클래스에 정의한 변수들
			- 멤버 변수(Member Variable)
				- 특정 클래스에 소송된 멤버 변수
			- 필드(Field)
				- 데이터 항목을 가르킴
				
3. 클래스와 사용자 정의 타입
	1) 타입은 데이터의 종류나 형태를 나타냄
	2) Product라는 타입을 만듦
	3) 사용자가 직접 정의하는 사용자 정의 타입 만들기 위한 설계도가 필요함
		- 이때의 설계도가 클래스임
	4) 설계도인 클래스를 사용해서 실제 메모리에 만들어진 실체를 객체 또는 인스턴스라 함
	
4. 클래스, 객체, 인스턴스 정리
	1) 클래스 (Class)
		- 클래스는 객체를 생성하기 위한 '틀' 또는 '설계도'임
		- 클래스는 객체가 가져야할 속성(변수)과 기능(메서드)를 정의함
		- 틀: 붕어빵 틀, 실제 먹을 수 있는 것이 아님
		- 설계도: 자동차 설계도, 설계도가 자동차는 아님. 개념으로만 있음
		
	2) 객체 (Object)
		- 클래스에서 정의한 속성과 기능을 가진 실체임
		- 서로 독립적인 상태를 가짐
	
	3) 인스턴스 (Instance)
		- 특정 클래스로부터 생성된 객체를 의미함.
		- 인스턴스는 객체보다 좀 더 관계에 초점을 맞춘 단어임
		
5. 객체 사용
	1) 클래스를 통해 생성한 객체를 사용하려면 먼저 메모리에 존재하는 객체에 접근해야 함.
		- 객체 접근하려면  .(점 doc)을 사용하면 됨.
	2) 객체에 값 대입
	3) 객체 값 읽기
