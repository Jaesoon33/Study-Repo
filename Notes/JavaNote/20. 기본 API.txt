1. java.lang 패키지
    1) import 없이 자동으로 사용 가능한 기본 패키지 
        - Object (모든 클래스의 최상위 부모 클래스)
            - 실무에서 반드시 재정의
                - equals() : == 과 동일 --> 내용(값) 비교로 변경
                - hashCode() 
                - toString() : 의미 있는 정보 출력 

            - equals()를 오버라이드하면 hashCode도 반드시 오버라이드해야 함
              equals가 true인 두 객체는 같은 hashCode를 가져야하는 규약 때문임.                

            - new 할 때 JVM이 하는 일
                - Heap 메모리에 공간 확보 
                - 필드 초기화 
                - hashCode 값 자동 부여 (메모리 주소 기반)   
                    - 예) 사람이 태어나면 --> 주민번호가 발급
                          객체가 new되면 --> hashCode가 자동 부여됨
                    - 객체마다 다른 숫자 
                    - m1 객체                  m2 객체
                      id = "A001"             id = "A001"   (같은 값)          
                      메모리 주소 0x7A3F        메모리 주소 0xB21E
                      hashCode()               hashCode() 
                       = 1523554304               = 789012345
                                    hashCode 다름!

            - 비교 
                구분     hashCode() 기준     m1 vs m2 (id같음)   equals()    결과
                ----------------------------------------------------------------------
                재정의X   메모리주소          hashCode 다름         호출안됨    못찾음(버그)
                재정의O   id필드 값          hashCode 같음          호출됨     찾음 (정상)

        - String
            - 불변성(immutable)과 메모리 구조 
                - 예
                    String s1 = "Hello";
                    String s2 = "Hello";
                    String s3 = new String("Hello")

                    s1 = s1 + " World";
                    // s1은 이제 새 객체를 참조
                    // 원래 "Hello"는 변경되지 않음
