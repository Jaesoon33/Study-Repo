1. 생성자 (constructor)가 없는 경우
	1) 회원 객체의 벰버 변수에 초기값을 성정함
		- 회원의 초기값 설정하는 부분이 계속 코드 중복됨
		--> 메서드를 이용하여 반복을 제거함
			- 해당 메서드를 객체지향 캡슐화하는 방법으로 사용함

2. 생성자가 필요한 이유
	1) 자기 자신의 데이터를 초기화하는
	
3.  생성자 특징
	1) 메서드와 비슷
	2) 생성자의 이름은 클래스 이름과 같아야 함

4. 생성자 호출
	1) 생성자는 인스턴스를 생성하고 나서 즉시 호출됨
	2) 생성사 호출 방법
			- new 클래스 이름(생성자에 맞는 인수 목록)
			- new 생성자 이름(생성자에 맞는 인수 목록)
			
			- new 키워드를 사용해서 객체를 생성할 때 마지막에 ()도 포함함
	3) 객체를 생성하면서 동시에 생성자를 호출함

5. 기본 생성자 (default constructor)
	1) 생성자를 만들지 않았는데, 생성자를 호출함
	2) 매개변수가 없는 생성자를 기본 생성자라 함
	3) 클래스에 생성자가 하나도 없으면 자바 컴파일러는 매개변수가 없고,
		작동하는 코드가 없는 기본 생성자를 자동으로 생성함
		  - 생성자가 없으면 기본 생성자가 제공됨
	4) 생성자가 하나라도 있으면 자바는 기본 생성자를 만들지 않음
	
6. 생성자의 제약
	1) 생성자 호출 필수
		- 객체를 생성하려면 생성자는 반드시 호출되어야 함
	2) 객체를 생성할 때 직접 정의한 생성자가 있다면
		- 직접 정의한 생성자를 반드시 호출해야 함
	3) 생성자를 메서드 오버로딩처럼 여러 개 정의할 수 있음
		- 객체 생성 시 여러 생성자 중 하나만 호출하면 됨
	4) 생성자를 사용하면 필수 값 입력을 보장할 수 있음
	
7. 생성자 오버로딩과 this()
	1) 생성자도 메서드 오버로딩처럼 매개변수만 다르게 해서 여러 생성자를 제공할 수 있음
	2) 생성자 코드가 중복되는 부분이 있음
	3) this()를 사용하면 생성자 내부에서 자신의 또 다른 생성자를 호출할 수 있음
	4) this()는 생성자 코드에서 첫줄에서만 작성이 가능
