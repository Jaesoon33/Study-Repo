1. 인터페이스란
	1) 목적
		- 서로 다른 클래스들이 같은 방식으로 동작하도록 표준화 
	2) 핵심
		- 추상 메서드만 정의 (구현은 각 클래스가 담당) --> 행위의 명세만 제공
		
2. 인터페이스의 활용
	1) 다형성 구현
		- 서로 다른 클래스들을 같은 타입으로 처리
	2) 느슨한 결합 (Loose Coupling)
		- 구현체가 아닌 인터페이스에 의존
		- 구현체를 쉽게 교체 가능
	3) 다중 상속의 대안
		- 클래스는 단일 상속(extends), 인터페이스는 다중 구현(implements)
	4) API/프레임워크 설계
		- 표준화된 API 제공
		- ex)
			- JDBC, JPA, Servlet API

4. 다중 인터페이스 구현
	1) 여러 인터페이스 동시 구현
	2) 시나리오
		- 기존 문서 출력 시스템을 확장해달라 요청
		- 출력 기능 외에 다음 기능들이 필요함
			- 저장(Save)
			- 공유(Share)
			- 검색(Search)
		- 단, 모든 문서가 모든 기능을 지원하는 것은 아님
			- Word 문서: 모든 기능 지원(출력, 저장, 공유, 검색)
			- PDF 문서: 읽기 전용이라 출력, 검색만 가능(저장, 공유 불가)
			- 메모: 개인 메모라서 저장, 검색만 가능(출력, 공유 불가)
	3) 인터페이스 분리 원칙(ISP: Interface Segregation Principle)
		- 하나의 큰 인터페이스 대신, 작은 인터페이스 여러 개로 분리함
		- 각 클래스는 필요한 인터페이스만 선택적으로 구현함
	4) 문법 
		- class 클래스명 implements 인터페이스1, 인터페이스2, ... { ... }
		- 쉼표로 여러 인터페이스를 나열함
		- 각 인터페이스의 모든 추상 메서드를 구현해야 함
