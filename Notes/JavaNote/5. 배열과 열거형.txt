1. 배열(Array)이란?
	1) 같은 타입의 데이터를 연속된 메모리 공간에 저장하는 저장구조임.

2. 배열 알고리즘
	1) 합계, 평균, 최대/최소값
	2) 선형 검색 (Linear Search)
		- 처음부터 끝까지 하나씩 비교
		- 장점: 정렬되지 않은 배열에서도 사용이 가능함
		  단점: 데이터가 많으면 느림
		- 사용 시기
			- 배열이 정렬되지 않았을 때
			- 배열 크기가 적을 때 (<1000)
			- 한 번만 검색할 때

3. 다차원 배열
	1) 2차원 배열 개념 
		++
		stack -> heap으로 전달될 때 (주소 참조)
		heap에 전달된 값이 주소 -> 값으로 나타남
		즉 참조가 stack과 heap에서 각각 1번씩 일어나
		총 2번의 주소 참조가 일어난 구조의 배열을 2차원 배열로 함
		- 배열의 배열
		- 행(row)과 열(column)으로 구성된 표 형태의 데이터를 저장함
	
	2) 메모리 구조
		- int[][] matrix = new int[3][4];
		
			matrix ------->      [참조0] 	   [참조1]		  [참조2]	(1차원 배열: 행 참조)
							       |	    	  |     	    |
							  [1][2][3][4]   [5][6][7][8]  [4][3][2][1]
							  
4. 배열 복사 (Array Copy)
	0) 실제 상황에서 필요한 경우
		- 원본 보전
			- 정렬/수정 전에 원본 데이터를 백업해야 할 때
			- 방어적 복사
				- 메서드에서 받은 배열을 수정해도 호출자에게 영향 없게
			- Undo 기능
				- 작업 전 상태를 저장해두고 되돌리기 구현
			- 병렬 처리
				- 여러 스레드가 독립적인 배열로 작업할 때
		- 원본
	
	1) 배열은 참조 타입이므로, 단순 대입(=)은 주소만 복사됨
		- 이건 복사가 아니라 같은 배열을 두 변수가 공유하는 형태라 볼 수 있음
		- "얕은 복사"가 문제가 되는 경우가 있고 아닌 경우도 있음
		
	2) 1차원 기본형 배열 (int[], double[] char[] ...)
		- "얕은 복사"로 충분함
		- 값 자체가 복사되므로 원본과 복사본이 완전 독립
		- 4가지 방법 사용 가능
		
5. 얕은 복사 VS 깊은 복사		

		구분			얕은 복사(Shallow)			깊은 복사(Deep)
	-------------------------------------------------------------------
	  복사 대상 		참조(주소)만 복사				실제 객체까지 모두 복사
	  원본 수정 시	복사본도 영향 받음				복사본은 영향 없음(완전 독립)
	  기본형 배열로	문제 없음(값도 복사됨)			필요 없음
	  객체 배열 		주의! 내부 객체 공유			안전(내부 객체도 새로 생성)
	  성능			빠름							느림 (객체 생성 비용)
	  메모리			적음							많음 (객체 복제)
	  
6. 열거형(Enum) 이란?
    1) 관련된 상수들의 집합을 정의하는 특별한 데이터 타입
    2) 미리 정해진 값들 중에서만 선택할 수 있게 제한함 
        - 정해진 값들만 가질 수 있는 특별한 타입 

7. 왜 Enum이 필요한가?
    1) 상수를 사용한 기존 방식(의 문제점) 
        - 예
            static final int MONDAY = 1;
            static final int TUESDAY = 2;   
            static final int WEDNESDAY = 3;
            //... 생략

            static final int JANUARY = 1;

    2) 사용하는 이유
        - 타입 안정성 -- 정해진 값만 사용 가능
        - 가독성 향상 -- 숫자 대신 의미 있는 이름 사용
        - switch문과 궁합이 좋음
