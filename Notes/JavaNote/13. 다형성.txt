1. 다형성(Polymorphis
	1) 하나의 타입, 여러 형태의 동
	2) Poly(많은) + morph(형태) = 많은 형태를 가진다
	3) 하나의 참조 변수(부모 타입)로 여러 타입의 객체(자식들)를 다룰 수 있는 노력
	4) 부모 타입 변수 하나로 모든 자식 객체를 담아 처리할 수 있음
	5) 다형적 참조
		- 다양한 형태, 여러 형태를 뜻함
		- 보통 하나의 객체는 하나의 타입으로 고정되어 있음
		- 다형성을 사용하면 하나의 객체가 다른 타입으로 사용될 수 있음
		
	- ex
		Shape[] shapes = new Shape[3];
		
		shapes[0] = new circle(5);			// 원 객체 (업캐스팅)
		shapes[1] = new Rectangle(4, 6)		// 사각형 객체(업캐스팅)
		shapes[2] = new Triangle(3, 4, 5)	// 삼각형 객체(업캐스팅)
		
2. 업캐스팅(Upcasting)
	1) 자식 ---> 부모: 부모 = 자식
	2) 자식 클래스 객체를 부모 클래스 타입의 변수에 저장하는 것
	3) 자동 변환(암묵적), 캐스팅 연산자 필요 없음
	4) 주의할 점
			- 부모 클래스에 정의된 멤버만 접근 가능(자식 고유 멤버 접근 불가)
			- 오버라이딩된 메서드는 자식의 것이 실행됨 (동적 바인딩)

3. 다운캐스팅(Downcasting)
	1) 부모 --> 자식 : 자식 = 부모
	2) 부모 타입 변수를 원래 자식 타입으로 되돌리는 것이
	3) 명시적 변화 필수, 캐스팅 연산자 필요
	4) 반드시 instatnceof로 확인 후 진행
	5) 왜 다운캐스팅이 필요한가
		- 업캐스팅 후 부모 타입으로는 자식 고유 멤버에 접근 불가!
		- 자식만의 특별한 기능을 사용하려면 원래 타입으로 되돌려야 함
	6) instanceof 연산자
		- 객체가 특정 클래스의 인스턴스인지 확인하는 연산자 (T/F 반환)
		- 잘못된 다운캐스팅 --> ClassCastException 발생  
		
4. 동적 바인딩 (Dynamic Binding)
		1) 메서드 호출이 실행 시점에 실제 객체의 타입에 따라 결정되는 것
		2) 컴파일 시점이 아닌 런타임에 어떤 메서드를 실행할지 결정 --> Late Binding이라고도 함
		3) 정적 바인딩 (static Binding) vs 동적 바인딩 
        - 정적 바인딩 
            - 결정 시점 : 컴파일 시 
                - 참조 변수 타입에 해당 메서드가 있는지 확인 (문법 검사)
            - 적용 대상 : private, static, final 메서드 
            - 오버로딩 
        - 동적 바인딩
            - 결정 시점 : 실행 시 (런타임)
                - JVM이 실제 객체 타입 확인 --> 오버라이딩된 메서드 호출 
            - 적용 대상 : 오버라이딩된 인스턴스 메서드
