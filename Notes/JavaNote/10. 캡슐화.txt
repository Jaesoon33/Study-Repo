1. 절차 지향 프로그래밍
	1) 절차를 지향함
	2) 실행 순서를 중요하게 생각하는 방식임
	3) 프로그램의 흐름을 순차적으로 따르며 처리하는 방식임
	4) "How"를 중심으로 프로그래밍 함
	
2. 객체 지향 프로그래밍
	1) 객체를 지향함
	2) 객체를 중요하게 생각하는 방식임
	3) 실제 세계의 사물이나 사건을 객체로 보고, 이러한 객체들 간의
	   상호 작용을 중심으로 프로그래밍하는 방식임
	4) "What"을 중심으로 프로그래밍 함

3. 두 방식의 중요한 차이
	1) 절차 지향은 데이터와 해당 데이터에 대한 처리 방식이 분리되어 있음
	2) 객체 지향은 데이터와 그 데이터에 대한 행동(메서드)이 하나의
	    "객체"안에 함께 포함되어 있음 --> 캡슐화(Encapsulation)
		
4. 캡슐화
	1) 객체 지향 프로그래밍의 중요한 개념 중 하나임
	2) 데이터와  해당 데이터를 처리하는 메서드를 하나로 묶어서
	   외부에서의 접근을 제한하는 것을 말함
	3) 캡슐화를 통해 데이터의 직접적인 변경을 방지하거나 제한할 수 있음
	4) 속성과 기능을 하나로 묶고, 외부에 꼭 필요한 기능만 노출하고
	   나머지는 모두 내부로 숨기는 것
	   
5. 접근 제한자(access modifier)
	1) 종류
		- private: 모든 외부 호출을 막음
				   클래스 내부로 속성과 기능을 숨길 때 사용
				   외부 클래스에서 해당 기능 호출 불가
		- (default): 같은 패키지 안으로 속성과 기능을 숨길  때 사용
					 외부 패키지에서 해당 기능 호출 불가
		- protected: 같은 패키지에서의 호출은 허용함
					 패키지가 달라도 상속 관계의 호출은 허용함
					 상속 관계로 속성과 기능을 숨길 때 사용
					 상속 관계가 아닌 곳에서 해당 기능 호출 불가
		- public: 모든 외부 호출을 허용
		
	2) private > (default) > protected > public 
		- 순서대로 private가 가장 많이 ㅏ단함
		
	3) 사용 위치
		- 필드, 메서드, 생성자에 사용됨
		- 클래스
			- public, (default)만 사용 가능함
			- private, protected는 사용 불가함
			- public 클래스는 반드시 파일명과 이름이 같아야함
