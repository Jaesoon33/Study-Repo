2. 추상화(Abstraction)
	1) 정의
		- 복잡한 시스템에서 핵심적인 개념만 추출하여 단순화
	2) 현실 세계의 자동차
		- 수천 개의 부품
		- 복잡한 엔진 구조
		- 전기 & 냉각 시스템 etc..
		- details..
	3) 프로그래밍 모델
		- 핵심 속성: speed, brand, fuel
		- 핵심 동작: start(), accelerate(), stop()
		- --> 필요한 것만 추출하여
		
3. OOP에서의 추상화 구현 방법
	1) 추상 클래스 (Abstract Class)
		- 부분적 구현 기능
		- 상태(필드) 보유
	2) 인터페이스 (Interface)
		- 순수한 명세, 구현 없음
		- 상수만 가능
		- 행위(능력) 정의 시 사용
		
4. 추상 클래스 (Abstract Class) 정의
	1) 인스턴스를 직접 생성할 수 없고, 상속을 통해서만 사용되는 클래스
	2) 특징
		- 필드 선언 가능
			- 일반, static 변수(Variable)을 멤버로 가질 수 있음
		- 생성자 정의 가능
		- 일반 메서드(구현 포함) 추가 가능
		- 추상 메서드(구현 없음) 가능
		- new로 직접 인스턴스 생성 불가!
			- 반드시 상속해서 사용되는
	3) 일반 클래스 vs 추상 클래스
	
			일반 클래스					추상 클래스
	-------------------------------------------------------
		new로 객체 생성 가능			new로 객체 생성 불가
		모든 메서드 구현 필수			추상 메서드 구현 생략 가능
		상속 - 선택 사항				상속 - 필수 사항 (To use)
		완전한 클래스					불완전한 클래스 (설계도)
	
5. 추상 메서드 (Abstract Method)
	1) 메서드의 시그니처(선언)만 있고, 구현부(body)가 없는 메서드
	2) ex
		- public abstract double getArea();
