1. 기본 이미지 삽입
    1) img 태그 기본 구조
        - <img src="이미지경로" alt="대체텍스트"> 
            - 반드시 src와 alt 속성을 포함해야 함
            - src 속성 (source)
                - 상대 경로 
                    - 현재 파일 위치 기준
                    - 예시) ../images/photo.jpg
                - 절대 경로
                    - 웹사이트 루트 기준 
                    - 예시) /images/photo.jpg
                - 외부 url

        - 닫는 태그가 필요 없는 빈 요소 (empty element)

    2) alt 속성 (alternative text)
        - 이미지를 표시할 수 없을 때 보여줄 대체 텍스트

        - alt 속성의 중요성
            - 시각장애인이 스크린 리더로 웹을 탐색할 때 이미지 내용 파악 가능
            - 이미지 로딩 실패 시 대체 텍스트 표시
            - 검색엔진(SEO)이 이미지 내용을 이해하는 데 도움
            - 웹 접근성 필수 요소 

        - 좋은 alt 작성요령
            - 이미지의 내용/목적을 간결하게 설명
            - "이미지", "사진" 같은 단어는 불필요
            - 장식용 이미지인 경우에는 alt="" (빈 값)
            
        - 이미지 나란히 배치 
            - <img> 태그는 기본적인 인라인(inline) 요소
                - 인라인 요소는 줄바꿈 없이 한줄에 나란히 배치
                - width 속성으로 이미지 크기 조절 (px 단위)
                    - height는 비율에 맞게 자동 계산됨
2. 이미지 크기 조절
    1) 픽셀(px) 단위
        - 특징
            - 고정 크기로 항상 같은 크기로 표시
            - 화면 크기/해상도와 무관하게 일정하게 표시
            - 모바일에서 너무 크면 가로 스크롤 발생 가능
        - 장점
            - 정확한 크기 제어 가능
            - 레이아웃 계산이 쉬움
            - CLS(레이아웃 이동) 방지 효과
        - 단점
            - 고정 크기라 반응형에 불리
            - 모바일에서 이미지가 잘릴 수 있음 
            - 고해상도 화면에서 흐릿할 수 있음

    2) 퍼센트(%) 단위
        - 특징
            - 부모 요소의 크기를 기준으로 계산
            - 50% = 부모 너비의 절반
                - 부모가 600px width이면 이미지는 300px width를 가짐

    3) 반응형 이미지 (CSS Class)                
        - .img-responsive {
            max-width: 100%;
            height: auto;    
        }
            - max-width: 100%;
                - 이미지가 부모 요소보다 커지지 않도록 제한(최대 너비 지정)
            - height: auto
                - 너비에 맞춰 높이 자동 계산(비율 유지)

    4) width vs max-width
        - width: 100%       항상 부모 너비에 맞춤(작은 이미지도 늘어남 => 화질 저하)
        - max-width: 100%   최대 부모 너비까지만, 원본보다는 안 커짐(화질 저하 없음)

3. figure와 figcaption
    1) figure 태그
        - 시맨틱(의미론) HTML5 태그
        - "독립적인 콘텐츠"를 나타냄
        - 이미지, 도표, 코드 블록, 인용문 등에 사용
    2) div vs figure
        - <div>     의미 없는 범용 컨테이너
        - <figure>  "이것은 독립적 콘텐츠"라는 의미 전달
    3) figure 사용 장점
        - 시맨틱 마크업으로 코드 가독성 향상
        - 스크린 리더가 "그림" 영역임을 알려줌
        - 검색엔진이 콘텐츠 구조를 더 잘 이해함 (SEO)
        - figure 단위로 스타일 적용 용이
    4) figcaption 특징
        - <figure> 내부에서만 사용 가능
        - figure 맨 앞 또는 맨 뒤에 위치
        - 하나의 figure에는 하나의 figcaption만 사용 가능
    5) alt vs figcaption
        - alt          이미지를 대체하는 텍스트 (이미지가 안 보일 때)
        - figcaption   이미지를 보충 설명하는 캡션 (항상 보임)
        - therefore 둘다 작성하는 것이 좋음

5. 이미지 스타일링
    1) 원형 이미지 (border-radius) 조건 
        - width = height (정사각형)
        - border-radius: 50% (정확한 원)
        - 참고
            - border-radius: 10px (살짝 둥근 모서리)
            - border-radius: 0    (직사각형)
        - 활용처 : 프로필 사진, SNS 아바타 
    
    2) 그림자 효과 (box-shadow)
        - box-shadow: values
            - ex)box-shadow: 0 4px 9px rgba(0,0,0,0.3)
            - Syntex element (1. 수평거리, - 수평으로 고정
                              2. 수직거리, - 아래로 4px 이동
                              3. 흐림정도, - 그림자 흐림정도
                              4. 색상    ) - 30% 투명도를 가진 검정색
        - 그림자 강도 조절
            - 은은한 : rgba(0,0,0,0.1)  10%
            - 보통 : rgba(0,0,0,0.2)    20%
            - 강한 : rgba(0,0,0,0.3)    30%

    3) 호버 효과 (transition + transform)
        - transform: scale() 값
            - 1 = 원본 크기 (100%)
            - 1.1 = 확대 (110%)
            - 0.9 = 축소 (90%)
            - width 대신 transform을 사용
                - transform은 GPU를 가속하여 사용 --> 더 부드러운 애니메이션 처리 가능
                - 레이아웃에 영향 안 줌 (주변 요소 안 밀림 : 커지더라도 주변에 영향 X)
                - 성능이 훨씬 좋음
    
6. CSS Grid 갤러리
    1) CSS Grid로 이미지 갤러리 만들기
        - grid-template-columns로 열 정의하기
        - gap으로 간격 설정하기
    
    2) CSS Grid 기본
        - 속성
            - display: grid (그리드 컨테이너 선언)
            - grid-template-columns (열(column)정의)

        - repeat() fr 단위
            - repeat(반복횟수, 크기)
                - ex) grid-template-columns: repeat(3, 1fr);
                    - 3 = 3번 반복 (3열)
                    - 1fr = 남은 공간을 균등 분배(fraction)
                    - 결과: 3개의 동일한 너비 열
            - fr
                - 사용 가능한 공간을 비율로 분배
                - ex) 1fr 1fr 1fr   ->  1:1:1 = 각각 33.33%
                      1fr 2fr 1fr   ->  1:2:1 = 각각 25%, 50%, 25%
