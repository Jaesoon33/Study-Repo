1. form 태그 주요 속성
    1) action
        - 폼 데이터를 전송할 서버 URL   
            - 현재 상태: 백엔드 서버가 없어서 404 에러 발생
            - 실제 서버 엔드포인트 URL로 연결
    2) method
        - 데이터 전송 방식
        - POST: 데이터가 URL에 노출되지 않음 (보안에 유리)
        - GET : 데이터가 URL에 표시됨 (?key=value)
    3) name
        - 폼의 이름 (JS에서 접근할 때 사용)
        - 현재 상태
            - [문의하기 버튼 클릭]
                => 브라우저가 /submit으로 POST 요청
                => 서버 없음 => 404 NOT Found : server가 없는 현재 상태
        - 백엔드 연동 시 상태
            - HTML form => /submit =>         백엔드 서버               ===>      DB 저장
               (프런트)      (URL)     (Java/Spring, Node.Js, Python)         (MySQL,Oracle)

2. 입력 필드
    1) label의 for 속성은 input 태그의 id 속성과 연결됨
        - 레이블 클릭 시 해당 입력 필드로 포커스 (접근성 향상)
    2) input
        - type="text"       : 한 줄의 텍스트 입력 필드
        - type="email"      : 이메일 형식을 자동으로 필수 (@의 포함 여부)
                              모바일에서 @ 키가 포함된 키보드가 표시됨
                              유효하지 않은 형식 입력 시 제출 차단
        - type="tel"        : 전화번호 입력 전용 필드
                              모바일 기기에서 숫자 캐피드가 자동으로 표시됨
                              형식 검증은 자동으로 되지 않음
        - type="radio"      : 같은 name을 가진 라디오 버튼 중 하나만 선택 가능
                              다른 옵션 선택/클릭 시 기존 선택된 라디오 버튼이 해제됨                          
        - type="checkbox"   : 여러 개를 동시에 선택할 수 있음
                              각 체크박스는 독립적으로 on/off
        - type="range"      : 슬라이더 - 시각적인 범위를 선택하는 UI를 제공함
                                - 드래그로 값을 선택할 수 있어 직관적임
                                - 속성 (min, max, value, step)                                                            
        - type="number"     : 숫자만 입력 가능한 필드임
                              위/아래 화살표로 값을 조절할 수 있음                                
                          
        - id            : 요소의 고유 식별자 (label의 for 속성과 연결(동일))
        - name          : 서버로 전송될 때 사용되는 데이터 키(key) - 서버에서 key vlaue를 기준으로 proccessing
        - required      : 필수 입력 항목 (Empty한 경우 제출 불가)
        - placeholder   : 입력 전에 보여지는 힌트 텍스트 (입력 시 사라짐)

3. 드롭다운 메뉴
    1) select 태그 
        - 드롭다운 목록
        - 여러 옵션 중 하나를 선택하는 UI
            - option 태그로 선택 항목을 정의
                - value 속성 : 서버로 전송되는 실제 데이터

4. 여러 줄의 텍스트 입력
    1) textarea
        - input과 달리 여러 줄의 텍스트를 입력받을 수 있음
        - rows : 기본으로 보이는 줄 수 (노핑)
        - cols : 기본으로 보이는 너비
        - 기본값은 태그 사이에 입력(value 속성 아님)

5. 제출 버튼 (button)
    1) 클릭 시 form의 action의 URL로 데이터가 전송됨
    2) type 속성
        - submit : 폼 제출 (default)
        - reset : 폼 초기화 (모든 입력값 삭제)
        - button : 기본 동작 없음 (Js로 기능 구현)

6. fieldset, legend        
    1) fieldset: 관련 폼 요소들을 시각적/의미적으로 그룹화
    2) legend: 그룹의 제목 (테두리 위에 표시)
    3) 접근성 향상: 스크린리더가 그룹 구조를 인식
